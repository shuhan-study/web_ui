# Project: Modernization

_Opened 2026-04-20 after `v0.5-p5-complete`._

## North star

Bring `web_ui` to current major versions of Next.js and React without
regressing the features shipped in P1–P5. Maintenance project, not a
feature project.

## Problem statement (for Path B intake)

> Upgrade `web_ui` from **Next.js 15 → 16** and **React 18 → 19** (paired
> natural bump — Next 16 ships with React 19 support). **Tailwind 3 stays.**
> **Prisma stays.** **shadcn hand-seeded components stay** unless React 19
> directly breaks them. Project is a grades dashboard with Prisma + SQLite
> + next-themes. Resolve any transitive dependency breakage. Verify
> `npm run build` and cold-start manual test pass. This is a maintenance
> project — no new features, no new pages, no data-model changes.

Path B's intake polecat should use the above as the authoritative scope.
Anything in P5's carry-forward list (deploy, navbar link, landing page,
stale subtitle, `force-dynamic`) is **out of scope** for this project —
that belongs to the next project ("Deploy").

## Scope

**In:**
- Next.js 15 → 16 (including config/API migrations)
- React 18 → 19 (including type updates, deprecated pattern removal)
- Any transitive dependency breakage caused by the two upgrades
- Verification: `npm run build`, `npm run dev`, cold-start manual test

**Out:**
- Tailwind 3 → 4 (deferred to future "Modernization v2" project)
- Prisma, shadcn, next-themes version bumps (unless they directly break)
- New features (deploy → separate project)
- New pages or data model changes
- Test infrastructure (still deferred)
- Bundle-size or performance optimization

## Decision log

### 2026-04-20 — Scope narrowed to Next + React only

**Original intent (when Path B was first attempted):** bump Next + React + Tailwind together.

**Decision:** drop Tailwind 3 → 4 from scope. Defer to a future "Modernization v2" project.

**Why:**
1. Tailwind 4 is a CSS engine rewrite (Lightning CSS, CSS-first config,
   no PostCSS by default). Different mental model from the JS framework
   bumps.
2. Next 16 supports Tailwind 3 fine; no blocker.
3. Bounded scope = sharper PRD = less drift during Path B.
4. Natural pairing: Next 16 is designed to ship with React 19.

**What's not free:** Tailwind 3 will eventually lose active development.
Maintenance debt accumulates. A follow-on bead for Tailwind 4 should be
filed at completion of v0.6 so it's visible but not pressing.

### 2026-04-20 — Project naming shifted from "P6" to intent-named

**Decided:** new projects are named by intent ("Modernization", "Deploy"),
not by P-number. P1–P5 was Path A's cadence; Path B produces larger
epics that don't map to P-numbers.

**Tag scheme:** `v0.X-<project>-complete` (e.g., `v0.6-modernization-complete`).
